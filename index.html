记广州快一点公司前端实习生面试

发表于 2015-12-21   |   分类于 diary   |  

2015年12月18日，风和日丽，正应正了那句话：“以景衬情”，预示着好事的降临~~
地铁3号线到5号线，兜兜转转从华农大正门到测绘大厦，接下面试的过程差不多三小时。
面试还是感觉很完整流畅的，笔试-面试-hr面
笔试的题目大致是这样的：

①写出宽高为30px的，中间矩形自适应的demo
②写出你知道的水平垂直居中的demo
③谈谈模块化的好处
④你知道的原型链
⑤跨域是什么，对应的解决方案
⑥http的响应头和请求头的内容

大约35分钟过去了，两位技术哥哥走来了，一位前端一位后台，前端的哥哥问的基本都是考卷的内容，倒是后台哥哥问“git”查看修改过的命令时心里毛了，平时只是简单的用下而已，没有怎么去用到实际去。心里想到git add搭上某个参数也行（具体哪个倒是忘了，平时自己自娱自乐基本也没用过…），就答了git add

……

于是乎在下周一就去了报道，开始了我新的征程 ~

事件委托

事件委托利用了事件冒泡,只指定一个事件处理程序,就可以管理某一类型的所有事件

问题一：什么是事件冒泡？
事件开始时有最具体的元素（文档中嵌套层次最深的那个结点）接收,然后逐级传播到较为不具体的结点（文档）

DOM事件流
事件捕获阶段(事件由document对象首先接收,再到事件的实际目标)
事件发生阶段
事件冒泡阶段(与事件捕获顺序相反)

发表于 2015-11-07   |   分类于 Javascript   |  

requireJS

requireJS的作用
  • 实现js文件的异步加载，避免一次性加载过多的js文件，网页失去响应
  • 管理模块之间的依赖性，便于代码的依赖和维护

使用前，需要加载require.js 和配置app.js，引入格式如下：

<script src="require.js" data-main="app.js"></script>
